Learning Objectives

After completing this unit, you'll be able to:
  • Define what a flow is and what it’s comprised of.
  • Describe what types of business processes you should use a flow for.
  • Build a flow that creates an account, a contact, and an opportunity.

Introduction to Visual Workflow, the Cloud Flow Designer and Flows

You may have heard several terms used interchangeably when referring to flows. Before we begin, let’s take a minute to define each of them and dispel some confusion.

  • Visual Workflow—the product that encompasses the process of designing, managing, and running flows. If you’re not sure about the difference between Visual Workflow and workflow rules, check out this help topic.
  • Cloud Flow Designer—a point-and-click tool that you can use to design flows.
  • Flow—an application that automates a process by collecting, updating, editing, and creating Salesforce data. Flows can execute logic, interact with the Salesforce database, call Apex classes, and guide users through screens for collecting and updating data. Most often, flows are built by using the Cloud Flow Designer.

In short, Visual Workflow is the product name, Cloud Flow Designer is the tool, and flows are the thing that you create.

Should I Create a Flow?

We recommend starting with the Process Builder whenever possible. The user interface is easier to use than the Cloud Flow Designer. The best way to figure out whether you can start with the Process Builder is to ask yourself, “Do I need to get information from a user?”

  • If your answer is Yes, use Visual Workflow.

    An example of a process that requires user interaction is a customer service call script. The process for helping a customer troubleshoot varies based on the product that they’re using, so the customer service representative needs to provide the product before he/she knows how to proceed.

  • If your answer is No (i.e. it can run completely in the background), start with the Process Builder. If your process is too complex for what the Process Builder allows you to do, then use Visual Workflow.

    For example, your process doesn’t require user interaction but it needs to evaluate the record against multiple levels of criteria. When a new contact is created or updated, make sure that the address is complete. If it isn’t complete, request the full address. However, some contacts have opted out of phone calls, so check whether the Do Not Call checkbox is selected. If it is selected, email the contact with a request for the full address. If it isn’t selected, create a task for the contact owner to follow up with the contact and update the address.

Getting to Know the Cloud Flow Designer

When you design flows, you work from the Cloud Flow Designer. For a quick tour of the user interface, watch this video.

If you don’t feel like watching a video, here’s a short summary.

  • Manage the flow while you’re designing from the button bar (1).
    • Test how the flow works as you’ve designed it with Run. Always save first, though. This button runs the last-saved version of the flow that you have open.
    • Return to Setup with Close.
    • Manage the flow’s properties, such as its name, description, and interview label, with Edit Properties.
    • Monitor the flow version’s status, as well as whether it has any errors or warnings, with the indicator on the top right corner.
  • The left-side panel (2) contains three tabs.
    • The Palette houses all the elements that you can add to your flow.
    • The Resources tab houses all the resources that you can add to your flow.
    • The Explorer shows all of the elements and resources that you’ve already added to this flow.
  • The diagram of your flow appears on the canvas (3).

Flow Building Blocks

Every flow is made up of the same three building blocks.

  • Elements (1) are what appear on the canvas. To add an element to the canvas, drag it there from the palette. Each element represents an action that the flow can execute, such as looking up a specific account (Record Lookup) or collecting input from flow users (Screen).
  • Connectors (2) define the path that the flow takes at runtime. They tell the flow which element is second, third, fourth, etc.
  • Resources (3) are containers that represent a given value, such as field values or formulas. You can reference resources throughout your flow. For example, look up an account’s ID, store that ID in a variable, and then update that account by referencing the stored ID.

Designing a Flow

For a quick introduction to the process of creating a flow, watch this video.

We’re going to walk you through designing a flow to streamline the process of creating accounts. Depending on your org’s page layouts, accounts can have lots of fields. All those fields can overwhelm sales representatives when they just want to quickly create an account with no bells and whistles.

Tip

  • Draw out your business process before you try to automate it. Doing so makes it easier to configure when you use one of our tools for process automation.
  • When you design a flow that creates records, know which of the object’s fields are required at the field level. The flow doesn’t check for fields required by a page layout. From the flow’s perspective, a field is required if it’s required by the API (standard fields) or has Required selected in the field definition (custom fields). If the flow doesn’t supply a value for a required field, the flow fails and the record isn’t saved. The Cloud Flow Designer doesn’t identify API-required fields.

    If you’re not sure which fields are required, you can look at the Object Reference for Salesforce and Force.com for standard fields and the object’s definition page for custom fields.

  1. From Setup, enter Flows in the Quick Find box, then select Flows, and then click New Flow.
  2. Add a screen, where you can collect information from the user who runs the flow.
    1. From the Palette, drag a Screen element onto the canvas.
    2. In the General Info tab, name it New Account.
    3. From the Add a Field tab, double-click Textbox and then double-click Number.
      A textbox and a number field appear in the preview pane on the right.
    4. Select each field in the preview pane and enter the following for each field’s label.
      • Textbox: Account Name
      • Number: Phone Number
    5. Click OK to return to the canvas.
  3. Create an account by using the values that the user entered for the screen.
    1. From the Palette, drag a Record Create element to your canvas.
    2. Name it Create Account.
    3. In Create, select Standard | Account.
      When you create other flows, you can always choose other objects.
    4. Map the screen input fields to account fields.
      For Field, select... For Value, select...
      Standard | Name Screen Input Fields | Account_Name
      Standard | Phone Screen Input Fields | Phone_Number
      Here’s how the Assignments section of your Record Create element looks.
    5. Click OK.
  4. Connect the two elements together by clicking the node at the bottom of the Screen element and dragging the connector to the Record Create element.
    As explained earlier, connectors tell the flow which element to go to next.
  5. Identify which element the flow should start at.
    Because the Record Create element uses the values entered in the screen’s fields, the Screen element must come first.
    1. Hover over the Screen element’s top right corner.
    2. Click .

  6. Save the flow, and name it Quick Account.

    The flow’s properties includes some other information, like Flow Type. Leave the defaults selected.

Testing Your Flow

You’ve created your first flow. Awesome! Now test the flow to make sure it’s working as expected.
  1. If you don’t still have the flow open, open it in the Cloud Flow Designer.
  2. In the button bar, click Run.
    An instance of the flow, which we call a flow interview, opens in another tab. You should see the first screen, which requests information for the account.
  3. Enter a name and phone number for a fake account, and then click Next.
  4. Close the tab.
  5. Verify that the test account was created.

Distributing Your Flow

Now that you’ve made sure the flow works as designed, distribute it to the right users.

Distribute Your Flow with Visualforce

Flows can be distributed in several ways. The Visual Workflow Guide goes into detail about all the ways you can distribute. Here, we’re going to walk you through just one: using a Visualforce page. When a flow is embedded in a Visualforce page, you can surface it just about anywhere. Some examples:
  • As a tab in a Force.com app
  • In a page layout
  • As a tab in a community or portal
  • In the Salesforce1 navigation menu

The word “Visualforce” might be intimidating, because it is a programmatic feature of Salesforce. But this is really easy. We’ll provide you with three lines of code, and you just have to change one thing to make your Visualforce page work. Then you’ll create a tab with that Visualforce page so that it shows up in the full Salesforce site.

  1. Find your flow’s unique name.
    1. If you aren’t already there, open the detail page for the Quick Account flow.
    2. Copy the flow’s Unique Name.
  2. Create a Visualforce page.
    1. From Setup, enter Visualforce Pages in the Quick Find box, then select Visualforce Pages. Then click New.
    2. Give your page a label.
    3. In the markup section, add the following code. 
      <apex:page>
   <flow:interview name = "YOUR_FLOW" />
</apex:page>
    4. Replace YOUR_FLOW with the unique name that you copied earlier.
    5. Click Save.
  3. Create a tab using that Visualforce page.
    1. From Setup, enter Tabs in the Quick Find box, then select Tabs.
    2. Under Visualforce Tabs, click New.
    3. For Visualforce Page, select the page you just created.
    4. Give your tab a label.
    5. Choose a tab style.
    6. Click Next, Next, and Save.
You should now see a tab called Quick Account (or whatever name you gave your tab). Click it to see your flow in action!

Ensure Users Can Run the Flow

Finally, make sure that your users can actually run the flow. For a user to be able to run this flow:
  • The user must have either the “Run Flows” user permission or Force.com Flow User selected on their user record.
  • The user must have access to the Visualforce page in which the flow’s embedded.
  • The flow must be active. Only flow creators (aka users with the “Manage Force.com Flow” user permission) can run inactive flows.
To address the first and second requirements, add the following things to a permission set and assign that permission set to the right users. For details about creating and assigning permission sets, see Controlling Access to Objects.
  • “Run Flows” user permission
  • The Visualforce page that you created

To address the third requirement, activate the flow. You must be on the flow’s detail page to do so.

  1. If you aren’t already on the flow’s detail page, navigate there.
    1. From Setup, enter Flows in the Quick Find box, then select Flows.
    2. Click the name of the flow that you need to activate.
  2. Activate the appropriate version of the flow.

Tell Me More...

More Flow Terminology

Here are a couple more terms that you should know when working with Visual Workflow.
  • A flow interview is a running instance of a flow. When you distribute a flow, users interact with individual interviews of that flow.
  • A flow can consist of multiple flow versions. Once you’ve activated a flow, you can’t make changes to it. You can, however, make the necessary changes to a new version of that flow and later activate the new version.

Doing More with Flows

This unit walked you through a simple example of a flow. You can customize that flow to do much more. For example, design the Quick Account flow so that it gives values for more account fields, like the location or owner.

Deleting Flows

To delete an active flow version, first deactivate it. If a flow has any paused or waiting interviews, it can’t be deleted until those interviews are finished or deleted. Flows that have never been activated can be deleted immediately. If you don’t see a Delete button or “Del” link, not enough time has passed since the flow was deactivated.
